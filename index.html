Customer experience

A 9-step wizard that just works

Guided activation from substrate to live service — SIM or eSIM — handling new numbers, porting, payments, identity verification, and consent.

1

Welcome

Branded landing page with your logo, copy, and link to your Critical Information Summary.

2

SIM & eSIM

Customers activate on SIM (validated ICCID against carrier inventory) or eSIM (digital profile path per your Octane/carrier setup). Backend supports QR-generation hooks for profile delivery where configured.

3

Number type

Customer chooses a new number or ports their existing one (prepaid or postpaid).

4

Account details

Full registration — name, DOB (18+ check), email, phone, Australian address, optional ABN for business.

5

Plan selection

Configurable upsell paths — e.g. international calling bolt-ons, data packs, premium tiers.

6

Number selection

New path: pick from available numbers. Port path: enter details, verify ownership via OTP.

7

Payment

Credit card or direct debit via Quickstream/Westpac trusted iframe. Tokenised. PCI-scoped.

8

Consent

T&C acknowledgement, porting authority, direct debit authorisation — plus a digital signature.

9

Confirmation

Order submitted. Success screen with order ID, activation timeline, billing info, and FAQ accordion — plus eSIM QR or install prompts when applicable.

Enterprise Security

Customer data protected by production-grade infrastructure

Your MVNO handles sensitive customer data — PII, payment tokens, porting credentials. The platform is built with security at every layer, from encrypted transport to isolated deployments.

🔒

PCI-compliant payment flow

Card data never touches your server. Quickstream's trusted iframe handles tokenisation directly with Westpac — your backend only receives a secure payment token reference.

🛡️

Isolated per-brand deployments

Each MVNO brand runs on its own deployment with separate environment credentials, API keys, and database rows. No shared state between brands.

Powered by Vercel & Replit
All checks passing
  • TLS 1.3 encryption on all endpoints
  • Edge DDoS protection enabled
  • Quickstream PCI DSS tokenisation
  • OTP-gated dashboard access
  • Environment-isolated API keys
  • Automated SSL certificate renewal

Encrypted end-to-end

Automatic HTTPS/TLS on every request. Customer PII and payment data are encrypted in transit from browser to backend — zero configuration required.

OTP-gated dashboard

Dashboard access requires email-verified one-time codes with rate limiting, cooldowns, and attempt caps. No passwords to leak or reset.

Credential isolation

Each brand fork uses its own Octane API credentials, SendGrid keys, Quickstream tokens, and Supabase project — stored in environment variables, never in code.

Full audit trail

Every activation, payment attempt, and order status change is logged with timestamps. Supabase records provide complete visibility for compliance.

Status alerting

Real-time email alerts when backend services degrade or go down. 15-minute cooldowns prevent alert storms while keeping your team informed.

Queue-based resilience

Order processing runs through a Redis-backed queue with automatic retries and dead-letter handling. Failed BSS calls don't lose customer data.
